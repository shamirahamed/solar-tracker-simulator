# Solar Tracker Simulator AI Context

Project:
Solar Tracker Simulator

Goal:
Build a web-based simulator for a single-axis solar tracker that calculates sun position, tracker rotation, shading, and estimated power generation for a full day.

Technology stack:

Backend
- Python
- FastAPI
- pvlib
- pandas
- numpy

Frontend
- HTML
- CSS
- JavaScript
- Canvas (2D animation)
- Chart.js

Repository structure:

backend/app/
  main.py
  models.py
  api/routes.py
  services/
    solar.py
    tracker.py
    shading.py
    power.py

frontend/
  index.html
  styles.css
  app.js

Simulation type:
Single-axis solar tracker
1-day simulation
1-minute resolution (1440 points)

Input parameters:

latitude
longitude
date
gcr
max_angle
backtracking

Calculation flow:

Location + Time
→ Solar position
→ Tracker rotation
→ Backtracking using GCR
→ Shadow length and shading
→ Irradiance
→ Estimated power

Output dataset per minute:

timestamp
sun_elevation
sun_azimuth
tracker_angle
backtracking_angle
shadow_length
shaded
power

Main API endpoint:

POST /api/v1/simulate/day

Response:
1440 rows of simulation data

Frontend behavior:

User enters parameters
→ API simulation request
→ backend returns daily dataset
→ frontend animates tracker using canvas
→ slider moves through the day
→ charts show tracker angle, sun elevation, power

Development rules:

- Keep backend modular
- Services handle calculations
- Routes only handle API
- Use pvlib for solar calculations
- Prefer simple implementations first
- Ensure JSON-friendly responses
